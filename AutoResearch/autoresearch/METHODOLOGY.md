# AutoResearch 研究方法论

基于 paper-research 的最佳实践，确保调研报告的学术规范性和高置信度。

## 核心原则

### 1. 信息可追溯性
- 每个重要观点必须标注来源
- 论文引用必须包含 arXiv 编号或 DOI
- 框架和工具必须提供官方链接
- 数据必须标注时间戳和来源

### 2. 学术规范性
- 使用标准的参考文献格式
- 区分预印本和正式发表论文
- 标注论文的 peer-review 状态
- 尊重版权，避免直接复制大段内容

### 3. 质量标准
- 来源可查：每篇论文标注来源链接
- 信息准确：摘要/结论需来自原论文
- 分类清晰：按主题/时间/方法分类
- 对比分析：横向对比不同论文/框架
- 趋势洞察：基于数据的趋势判断

## 研究方法论

### Phase 1: 需求确认

明确调研目标：
- 研究主题/关键词
- 需要什么类型的论文（理论/应用/综述）
- 时间范围要求
- 需要多少篇核心论文

### Phase 2: 智能规划

使用 LLM 分析用户需求，生成研究计划：
1. 提炼核心研究主题
2. 确定研究类型（学术/技术/市场/产品/通用）
3. 选择研究模式（single/deep/interactive）
4. 分解研究维度（3-6个关键维度）
5. 确定研究深度（basic/standard/comprehensive）

### Phase 3: 信息检索

**检索策略：**
- 使用 WebSearch 获取最新信息
- 优先查找 arXiv 预印本
- 查找官方文档和 GitHub 仓库
- 交叉验证信息来源

**来源优先级：**
1. arXiv - 最新预印本，速度快
2. 官方文档 - 权威性高
3. GitHub - 可复现性
4. 技术博客 - 实践经验

### Phase 4: 信息筛选与验证

**筛选标准：**
1. 相关性 - 与主题的直接关联度
2. 质量 - 期刊/会议级别
3. 时效性 - 最近3-5年为主
4. 可获取性 - 是否有链接/代码

**验证方法：**
- 交叉验证多个来源
- 检查论文的 peer-review 状态
- 确认链接的有效性

### Phase 5: 分析整理

对筛选后的信息进行深度分析：
1. 核心贡献提取
2. 方法论特点
3. 实验设置与结果
4. 优点与局限性
5. 与其他工作的关系

### Phase 6: 报告生成

生成结构化报告，包含：
1. **执行摘要** - 1-2段概括调研目的、主要发现、核心论文
2. **领域概述** - 研究背景、意义
3. **核心论文分析** - 按重要性排序，包含完整引用信息
4. **技术路线演进** - 时间线、对比分析
5. **研究趋势** - 年度发表量、关键词热度、机构分布
6. **实践建议** - 具体的应用建议
7. **参考文献** - 标准格式的完整参考文献列表

## 报告质量标准

### 信息完整性

每篇论文引用必须包含：
```
论文标题 (英文)
- 作者: First Author et al.
- 来源: arXiv:XXXX.XXXXX | 会议/期刊名 (年份)
- 链接: https://arxiv.org/abs/XXXX.XXXXX
- 核心贡献: 1-2句话
```

### 框架/工具信息必须包含：
```
框架名称
- 官方文档: https://...
- GitHub: https://github.com/...
- 核心特性: 3-5个要点
```

### 参考文献格式

使用统一的参考文献格式：

**论文引用：**
```
序号. 作者. 论文标题. 来源. 链接
```

**示例：**
```
1. Xu, R., & Yan, Y. Agent Skills for Large Language Models: Architecture, Acquisition, Security, and the Path Forward. arXiv:2602.12430. https://arxiv.org/abs/2602.12430
```

**网页引用：**
```
序号. 标题. 网站/组织. URL. 访问日期
```

### 对比分析表格

关键技术/框架对比应包含：
| 维度 | 选项A | 选项B | 对比说明 |
|------|-------|-------|---------|
| 核心特点 | ... | ... | ... |
| 优势 | ... | ... | ... |
| 局限 | ... | ... | ... |
| 适用场景 | ... | ... | ... |

## 置信度强化机制

### 来源标注

- **高置信度**: 有 arXiv/DOI 链接的论文
- **中置信度**: 官方文档/技术博客
- **低置信度**: 未经验证的信息（需标注）

### 时间戳

所有信息必须标注时间：
- 论文发表日期
- 报告生成日期
- 最后更新日期

### 更新机制

建立定期更新机制：
- 每季度更新一次综述类报告
- 重大会议（NeurIPS/ICML/ICLR）后更新
- 发现重要新论文时及时更新

## 注意事项

1. **优先使用官方来源**（arXiv、publisher官网）
2. **区分预印本和正式发表论文** - 标注 peer-review 状态
3. **注意论文的 peer-review 状态**
4. **标注可能存在利益冲突**
5. **尊重版权** - 不直接复制大段内容
6. **对于争议性结论** - 多方验证
7. **链接有效性** - 定期检查链接是否有效

## 输出标准

最终交付：
1. **Markdown 格式报告** (.md) - 主要格式
2. **完整的参考文献列表** - 按重要性排序
3. **资源链接汇总** - GitHub、文档、工具
4. **可视化图表** (如需要) - 趋势图、对比表

## 持续改进

根据使用反馈持续优化：
- 收集用户对报告质量的反馈
- 分析参考文献的引用率
- 跟踪研究领域的最新进展
- 更新研究方法论
