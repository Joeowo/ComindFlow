# RAG 技术深度调研报告

## 1. 概述与动机

**检索增强生成 (Retrieval-Augmented Generation, RAG)** 是一种结合信息检索与文本生成的技术范式，旨在解决大型语言模型 (LLM) 的固有局限：知识截止日期、幻觉 (hallucination)、缺乏特定领域知识和难以追踪信息来源等问题。RAG 通过在生成过程中动态检索外部知识库中的相关信息，将其作为上下文注入提示，从而提升生成内容的**事实准确性、时效性和可解释性**。

RAG 的核心优势：
- **事实基础**：用检索到的证据支撑生成内容，减少幻觉。
- **知识更新**：无需重新训练模型即可使用最新信息。
- **领域适应**：可接入专业数据库、企业文档等私有数据。
- **可解释性**：可通过检索来源追溯生成依据。

## 2. RAG 发展历程与范式演进

根据 Gao et al. (2024) 的综述，RAG 的发展可分为三个阶段：

| 阶段 | 代表工作 | 核心思想 | 局限性 |
|------|----------|----------|--------|
| **朴素 RAG** | Lewis et al. (2020) | 简单的“检索-读取”管道 | 检索质量不稳定，生成与检索脱节 |
| **高级 RAG** | Self-RAG, RR | 引入重排序、查询重构、自适应检索 | 流程固定，缺乏灵活组合 |
| **模块化 RAG** | ReAct, RAPTOR, GraphRAG | 可重组的模块与多步推理、图结构 | 系统复杂，编排挑战大 |

### 2.1 奠基之作

#### Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks
- **来源**: arXiv:2005.11401 (2020)
- **作者**: Patrick Lewis et al. (Meta AI / Facebook AI)
- **链接**: https://arxiv.org/abs/2005.11401
- **核心贡献**: 首次提出 RAG 框架，将预训练的密集检索器 (DPR) 与 BART 生成模型结合，通过端到端微调实现开放域问答。模型在生成时动态查询维基百科，显著提升了知识密集型任务的性能。

### 2.2 高级 RAG 的演进

#### Self-RAG: Learning to Retrieve, Generate, and Critique through Self-Reflection
- **来源**: arXiv:2310.11511 (2023)
- **作者**: Asai et al.
- **链接**: https://arxiv.org/abs/2310.11511
- **核心贡献**: 在 LM 中训练单个模型自适应地决定何时检索，并对自己生成的片段进行“反思”和批评，生成反思标记（`Retrieve`, `IsRel`, `IsSup` 等），从而在推理时动态控制检索行为，提高事实正确性与引用依据。

#### RAPTOR: Recursive Abstractive Processing for Tree-Organized Retrieval
- **来源**: arXiv:2401.18059 (2024)
- **作者**: Sarthi et al.
- **链接**: https://arxiv.org/abs/2401.18059
- **核心贡献**: 提出递归地将文档聚类并通过 LLM 生成摘要，构建层次化的树结构索引，支持在不同抽象层级上进行检索，从而能回答需要全局视角的总结性问题。

#### GraphRAG: Unsupervised Approach for Enhancing LLM with Knowledge Graphs
- **来源**: 微软技术报告 (2024)，无官方 arXiv，但相关方法在 GraphRAG 论文中描述
- **代表开源实现**: https://github.com/microsoft/graphrag
- **核心贡献**: 利用 LLM 从原始文本自动构建知识图谱，将实体和关系作为检索单元，生成社区摘要，支持全局问答。在数据发现、摘要等任务上优于常规 RAG。

## 3. 关键组成与增强技术

### 3.1 索引与分块

文档索引是 RAG 的基础，关键设计包括：
- **分块策略**: 固定大小分块、句子分块、语义分块（如基于句子嵌入相似度切割）。LangChain 提供了多种分块器。
- **多粒度索引**: 如 RAPTOR 的递归摘要树、精细块与大跨度摘要结合。
- **元数据与层级**: 保留标题、章节等结构信息以辅助检索。

### 3.2 检索技术

- **稀疏检索**: BM25 (基于 TF-IDF)，高效且可解释，但对语义理解弱。
- **密集检索**: 使用双编码器将查询和段落嵌入向量空间，代表为 DPR。可采用微调嵌入模型（如 `bge`, `voyage`, `text-embedding-3`）。
- **混合检索**: 融合稀疏与稠密检索结果，常见做法为加权求和或倒数排名融合 (RRF)。
- **多跳检索**: 针对需要链式推理的复杂问题，迭代检索，如 IRCoT 和 ReAct 模式。

#### Dense Passage Retrieval for Open-Domain Question Answering
- **来源**: arXiv:2004.04906 (2020)
- **作者**: Karpukhin et al.
- **链接**: https://arxiv.org/abs/2004.04906
- **核心贡献**: 提出密集段落检索器 (DPR)，使用双塔 BERT 结构，在维基百科上训练，极大推动了开放域问答中的密集检索应用。

### 3.3 查询增强

- **查询重写**: 用 LLM 重新表述用户查询以匹配文档风格，或分解复杂问题为子问题。
- **HyDE (Hypothetical Document Embeddings)**: 首先生成假设性回答，再用该回答的嵌入去检索相似文档。
- **查询扩展**: 使用同义词、额外上下文扩展查询。

#### Query2doc: Query Expansion with Large Language Models
- **来源**: arXiv:2303.07678 (2023)
- **作者**: Wang et al.
- **链接**: https://arxiv.org/abs/2303.07678
- **核心贡献**: 使用 LLM 生成伪文档，然后将原始查询与伪文档拼接后进行检索，提升稠密检索效果。

### 3.4 重排序 (Reranking)

在初步检索后，使用更精细的模型对候选文档进行重排序，筛选最相关片段。常用重排序模型：
- **Cross-encoder 模型**: 如 Cohere Rerank, `bge-reranker`, `cross-encoder/ms-marco-MiniLM-L-6-v2`
- **基于 LLM 的重排序**: Permutation generation, GPT-based scoring

#### Improving Passage Retrieval with Zero-Shot Question Generation
- **来源**: arXiv:2204.07496 (2022)
- **作者**: Sachan et al.
- **链接**: https://arxiv.org/abs/2204.07496
- **核心贡献**: 提出使用 LLM 生成问题-段落对进行数据增强，并利用交叉注意力重排序器提升检索相关性。

### 3.5 上下文整合与生成控制

- **上下文压缩**: 对检索到的长文档进行摘要或提取关键片段，避免超越上下文窗口。
- **系统提示词设计**: 明确要求模型基于检索信息回答，阻止编造。
- **过程反馈**: 如 Self-RAG 中的反思标记，或在生成时进行事实核查。

## 4. 评估体系

### 4.1 常用基准

| 基准 | 任务类型 | 描述 |
|------|----------|------|
| **Natural Questions (NQ)** | 开放域问答 | 用户向 Google 提出的真实问题，答案在维基百科中 |
| **TriviaQA** | 开放域问答 | 琐事问题，支持多段落证据 |
| **MS MARCO** | 文档排序 + 问答 | 大规模搜索日志查询，需为答案排序 |
| **HotpotQA** | 多跳推理 | 需要跨多个文档进行推理 |
| **MuSiQue** | 多跳推理 | 精心构建的多跳问题，涉及2-4个条目的推理 |
| **RAGAS** | 无参考评估 | 专用于评估 RAG 系统的指标框架 |
| **KILT** | 知识密集型任务 | 聚合多个知识驱动的任务基准 |

#### RAGAS: Automated Evaluation of Retrieval Augmented Generation
- **来源**: arXiv:2309.15217 (2023)
- **作者**: Es et al.
- **链接**: https://arxiv.org/abs/2309.15217
- **核心贡献**: 提出无参考评估方法，通过忠实度 (faithfulness)、答案相关性 (answer relevancy)、上下文相关性 (context relevancy) 等指标自动化评估 RAG 输出，无需人工标注。

### 4.2 评价维度

- **忠实度 (Faithfulness)**: 生成答案是否完全基于提供的上下文。
- **答案相关性 (Answer Relevance)**: 答案是否回答所提问题。
- **上下文相关性 (Context Relevance)**: 检索到的上下文是否与问题相关。
- **上下文召回率 (Context Recall)**: 答案所需信息是否被检索到。
- **多跳检索准确性**: 是否正确选择了必要的证据链。

## 6. 前沿方向与挑战

### 6.1 高级 RAG 架构

- **Self-Reranking 和自适应检索**: 学习何时检索、何时依赖内部知识，见 Self-RAG (arXiv:2310.11511)。
- **递归与抽象检索**: RAPTOR (arXiv:2401.18059) 的树结构，支持多层级信息聚合。
- **图结构 RAG**: GraphRAG 利用知识图谱中的关系推理，增强复杂查询下的表现。
- **多模态 RAG**: 扩展到图像、表格、代码等，使用多模态嵌入。

#### REVEAL: Retrieval-Augmented Visual-Language Pre-Training with Multi-Source Multimodal Knowledge Memory
- **来源**: arXiv:2312.05231 (2023)
- **作者**: Hu et al.
- **链接**: https://arxiv.org/abs/2312.05231
- **核心贡献**: 将 RAG 扩展到视觉语言模型，引入多源知识记忆库，可检索图像、文本、知识图谱等多模态知识。

### 6.2 代理与多步推理

结合代理框架，使 RAG 具备规划、工具使用和多步推理能力。

#### ReAct: Synergizing Reasoning and Acting in Language Models
- **来源**: arXiv:2210.03629 (2022)
- **作者**: Yao et al.
- **链接**: https://arxiv.org/abs/2210.03629
- **核心贡献**: 提出交替生成推理痕迹和行动的范式，使 LLM 能与外部环境（如检索系统）交互，实现多跳推理和动态检索。

### 6.3 主要挑战

1. **检索质量瓶颈**: 不完整的语义召回、噪声引入。
2. **长上下文整合**: 当检索大量文档时，如何进行有效的上下文选择与压缩。
3. **速度与成本**: 检索和生成引入额外延迟和计算开销。
4. **评估困难**: 缺乏全面、可跨系统比较的自动评估指标。
5. **领域适应性**: 通用嵌入在特定领域可能效果不佳，需要微调。
6. **信息冲突与一致性**: 当检索到的信息彼此矛盾或与模型内部知识冲突时，如何合理整合。
7. **安全与隐私**: 检索私有数据时的权限控制，避免泄露。

#### CRAG: Comprehensive RAG Benchmark
- **来源**: arXiv:2406.04744 (2024)
- **作者**: Yang et al.
- **链接**: https://arxiv.org/abs/2406.04744
- **核心贡献**: 提出了一个综合性的 RAG 评估基准，包含多类测试场景，并发现大多数 RAG 系统在处理“虚假信息”和“信息缺失”等微妙情况下的表现仍有巨大提升空间。

## 7. 实践建议与趋势总结

- **从简开始**: 采用朴素 RAG 管道（如 LangChain + Chroma），快速验证基线。
- **混合检索 + 重排序**: 这是当前生产环境中性价比最高的增强方案。
- **关注评估**: 使用 RAGAS 或类似工具持续监控检索和生成质量。
- **利用代理架构**: 对于复杂查询，引入生成-检索-推理的循环，如 ReAct 模式。
- **考虑专用模型**: 微调嵌入模型和重排序器以适配领域数据，或使用 Self-RAG 的思想训练自反思能力。
- **多模态与结构化数据融合**: 根据数据类型选择合适的检索单元（文本、表格、图等）。

RAG 作为增强 LLM 事实性和专业性的主流技术，正朝着 **更模块化、智能化、多模态** 的方向快速演进。预计未来将深度融合代理能力与知识图谱，并出现更多针对幻觉降低和长上下文理解优化的专有模型。
