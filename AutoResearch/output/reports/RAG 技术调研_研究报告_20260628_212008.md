# RAG 技术深度调研报告

## 1. 什么是检索增强生成 (RAG)

检索增强生成（Retrieval-Augmented Generation, RAG）是一种将**信息检索**与**文本生成**深度结合的技术范式。它通过在生成过程中动态检索外部知识库中的相关文档，将检索到的证据与用户查询一起送入大语言模型（LLM），从而缓解大模型的“幻觉”问题，提升生成内容的准确性、时效性和可验证性。

核心思想最早在以下论文中系统提出：

#### Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks
- **来源**: NeurIPS 2020, arXiv:2005.11401v4
- **作者**: Patrick Lewis, Ethan Perez, Aleksandra Piktus et al.
- **链接**: https://arxiv.org/abs/2005.11401
- **核心贡献**: 首次将预训练的检索器（Dense Passage Retrieval）与预训练的生成器（BART）端到端联合训练，在开放域问答等知识密集型任务上显著优于纯参数化模型，奠定了RAG范式的基础。

此后，RAG 快速演化，形成了“检索-阅读-生成”的基本框架，并催生出大量变体与优化方法。

## 2. RAG 的核心工作流程

一个标准的 RAG 系统包含三大阶段：

1. **离线索引**：将外部知识库（文档、网页、数据库等）分割成块（chunk），通过嵌入模型转化为向量，存入向量数据库，并建立文本/关键词索引。
2. **在线检索**：接收用户查询后，将查询映射到同一向量空间，从向量数据库中召回 Top-K 相关文档块；可选地结合关键词检索（BM25）进行混合检索。
3. **增强生成**：将检索到的文档与原始查询拼接成提示（prompt），引导 LLM 基于提供的证据生成最终回答，通常还会要求模型指明引用来源。

```
┌──────────────┐    ┌──────────────┐    ┌────────────────┐
│  离线索引阶段   │    │  在线检索阶段   │    │   增强生成阶段   │
└──────────────┘    └──────────────┘    └────────────────┘
 1. 加载文档         　　 　　　　　　　　　　　 6. 构建提示
 2. 文本分割              4. 查询向量化  　　 7. LLM 生成
 3. 向量化入库  ─────►  5. 相似度搜索  ─────► 8. 后处理与引用
```

## 3. 关键技术组件与代表性工作

### 3.1 检索器：从 Sparse 到 Dense，再到多向量与 Rerank

早期 QA 系统依赖 **BM25**（稀疏检索）。现代 RAG 普遍采用**稠密检索**（Dense Retrieval），核心是双塔模型（Query encoder + Document encoder）。以下是里程碑式工作：

#### Dense Passage Retrieval (DPR) for Open-Domain QA
- **来源**: EMNLP 2020, arXiv:2004.04906v3
- **作者**: Vladimir Karpukhin, Barlas Oğuz, Sewon Min et al.
- **链接**: https://arxiv.org/abs/2004.04906
- **核心贡献**: 提出使用两个独立的 BERT 编码器分别编码问题和段落，通过对比学习训练，显著提升了开放域问答的检索召回率，成为稠密检索的基石。

#### ColBERT: Efficient and Effective Passage Search via Contextualized Late Interaction over BERT
- **来源**: SIGIR 2020, arXiv:2004.12832v2
- **作者**: Omar Khattab, Matei Zaharia
- **链接**: https://arxiv.org/abs/2004.12832
- **核心贡献**: 在 token 级别进行迟交互（MaxSim），保留了细粒度语义匹配，同时通过离线预计算文档 embedding 实现了高效的在线检索，平衡了效果与效率。

#### SPLADE v2: Sparse Lexical and Expansion Model for Information Retrieval
- **来源**: SIGIR 2022, arXiv:2109.10086v3
- **作者**: Thibault Formal, Carlos Lassance, Benjamin Piwowarski, Stéphane Clinchant
- **链接**: https://arxiv.org/abs/2109.10086
- **核心贡献**: 利用 MLM 头实现稀疏、可学习的词汇扩展检索，效果逼近稠密检索，同时保留了稀疏检索的许多优点（可解释性、倒排索引）。

#### Cohere Rerank / bge-reranker
尽管不是严格意义上的论文，Reranker 已成为现代 RAG 的标准组件。在召回阶段后，使用 Cross-encoder（如 BERT）对 Top-N 文档进行精细重排序，可大幅提升最终文档的正相关率。代表性模型有 Cohere Rerank 和 BAAI 的 `bge-reranker-v2-m3`。

### 3.2 生成策略：上下文融合、归因与对抗幻觉

如何让 LLM 有效利用检索到的证据是关键问题。研究方向包括：

#### In-Context RALM (Retrieval-Augmented Language Model)
- **来源**: arXiv:2302.00083v1 (2023)
- **作者**: Ori Ram, Yoav Levine, Itay Dalmedigos et al.
- **链接**: https://arxiv.org/abs/2302.00083
- **核心贡献**: 通过在推理时简单地将检索到的文档拼接到 LM 输入，无需任何训练即可显著降低困惑度，系统研究了文档位置、检索频率等因素的影响。

#### Self-RAG: Learning to Retrieve, Generate, and Critique through Self-Reflection
- **来源**: arXiv:2310.11511v2 (2023)
- **作者**: Akari Asai, Zeqiu Wu, Yizhong Wang, Avirup Sil, Hannaneh Hajishirzi
- **链接**: https://arxiv.org/abs/2310.11511
- **核心贡献**: 训练 LM 自主按需检索，生成同时输出反思令牌（Reflection tokens）评估每个片段的相关性、支持性和完整性，实现了更可控的生成与更少的幻觉。

#### RAGAS: Automated Evaluation of Retrieval Augmented Generation
- **来源**: arXiv:2309.15217v1 (2023)
- **作者**: Shahul Es, Jithin James, Luis Espinosa Anke, Steven Schockaert
- **链接**: https://arxiv.org/abs/2309.15217
- **核心贡献**: 提出无需人工标注的 RAG 评估框架，涵盖忠实度、答案相关性、上下文召回率等指标，为 RAG 系统的自动化离线评估提供了标准化的工具。

### 3.3 知识库优化：Chunking、元数据与多模态

检索质量高度依赖上游的数据预处理。

- **Chunk 策略**: 固定长度分割、递归字符分割、基于文档结构（如 Markdown、HTML）的分割，以及“父子文档”策略（small chunks for retrieval, big chunks for generation）。
- **元数据过滤**: 利用 Self Query Retrieval (LangChain 概念) 或 llama-index 中的 Metadata filters，在检索时结合语义和结构化属性（时间、来源、标签）过滤。
- **多模态 RAG**: 将表格、图像作为检索对象。

#### Dense X Retrieval: What Retrieval Granularity Should We Use?
- **来源**: EMNLP 2023, arXiv:2312.06648v2
- **作者**: Yunfan Gao, Yun Xiong et al.
- **链接**: https://arxiv.org/abs/2312.06648
- **核心贡献**: 提出以命题（proposition）为检索粒度的方案，将段落分解为原子事实进行索引，有效提升了检索密度和下游 QA 的准确率。

## 5. 面临的主要挑战与解决方案

### 5.1 检索质量瓶颈

**问题**: 查询与文档的语义匹配不精确、知识库噪声导致检索到无关内容。

**解决方案**:
- **混合检索**：稠密向量 + BM25 关键词检索结合。
- **重排序**：Cross-encoder 模型对 Top-K 结果重新打分。
- **查询重写**：利用 LLM 将用户模糊查询转化为多个更利于检索的子查询或假设文档（HyDE）。

#### Hybrid Search Guide (Pinecone)
- **来源**: Pinecone 技术文档，非论文但对工业实践影响深远
- **链接**: https://www.pinecone.io/learn/hybrid-search-intro/
- **核心观点**: 稠密检索擅长语义扩展和长尾概念，稀疏 BM25 对精确术语匹配（如实体名、ID）更有效，二者互补可带来 10-20% 的召回提升。

#### Precise Zero-Shot Dense Retrieval without Relevance Labels (HyDE)
- **来源**: ACL 2023, arXiv:2212.10496v1
- **作者**: Luyu Gao, Xueguang Ma, Jimmy Lin, Jamie Callan
- **链接**: https://arxiv.org/abs/2212.10496
- **核心贡献**: 提出先由 LLM 生成一个假设答案文档（HyDE），再将该假设文档的 embeddings 用于检索，缩小了查询和文档的分布差异，零样本检索效果优异。

### 5.2 上下文过长与冗余

**问题**: 检索内容过多导致 LLM 的“迷失中间”（Lost in the Middle）问题，关键信息被淹没。

**解决方案**:
- **文档压缩**: LongLLMLingua 等方法压缩提示中的文档。
- **策略性排序**: 将最相关的文档放在输入序列的头尾位置。
- **多步推理**: IRCoT 等将检索过程与推理链交织进行。

#### Lost in the Middle: How Language Models Use Long Contexts
- **来源**: TACL 2023, arXiv:2307.03172v3
- **作者**: Nelson F. Liu, Kevin Lin, John Hewitt et al.
- **链接**: https://arxiv.org/abs/2307.03172
- **核心贡献**: 实证发现当关键信息位于长输入的中部时，LLM 的表现会急剧下降，为 RAG 的上下文编排提供了重要指导。

#### LLMLingua: Compressing Prompts for Accelerated Inference
- **来源**: EMNLP 2023, arXiv:2310.05736v3
- **作者**: Huiqiang Jiang, Qianhui Wu, Chin-Yew Lin, Yuqing Yang, Lili Qiu
- **链接**: https://arxiv.org/abs/2310.05736
- **核心贡献**: 提出了一种小语言模型指导的提示压缩方法，可在几乎不损失性能的前提下移除 20%-50% 的冗余 token，适用于 RAG 中的文档压缩。

### 5.3 多跳推理与复杂查询

**问题**: 需要综合多个文档中分散信息才能回答的复杂问题（多跳推理）。

**解决方案**:
- **迭代检索生成**: 根据已生成内容发起新检索，如 RETRO, IRCoT。
- **图结构索引**: Microsoft GraphRAG 利用知识图谱提取实体和关系，构建社区的层级摘要，解决全局性问题。

#### Interleaving Retrieval with Chain-of-Thought Reasoning for Knowledge-Intensive Multi-Step Questions (IRCoT)
- **来源**: ACL 2023, arXiv:2212.10509v3
- **作者**: Harsh Trivedi, Niranjan Balasubramanian, Tushar Khot, Ashish Sabharwal
- **链接**: https://arxiv.org/abs/2212.10509
- **核心贡献**: 交替进行思维链（CoT）生成和知识检索，用检索来信息化推理步骤，用推理步骤来指导下一步检索，在多步 QA 中大幅提升了效果。

#### GraphRAG: Unlocking LLM Discovery via Graph-Based RAG
- **来源**: 微软研究，2024年，GitHub 开源项目
- **链接**: https://microsoft.github.io/graphrag/
- **核心贡献**: 利用 LLM 从原始文本构建知识图谱，然后提取社区摘要，在查询时使用图社区摘要作为上下文，擅长回答摘要级、连接性等“全局”问题，与传统 RAG 形成互补。

## 6. 评估方法与基准

评估 RAG 系统需要同时评价检索质量和生成质量。常用基准与指标：

- **检索**: MRR, Recall@K, NDCG；基准有 Natural Questions (NQ), TriviaQA, MS MARCO, BEIR。
- **端到端 RAG**: 忠实度 (Faithfulness)、答案相关性 (Answer Relevance)、上下文相关性 (Context Relevance)；基于 LLM 的评测（如 RAGAS, ARES）逐渐成为主流。

#### ARES: Automated Retrieval Score for Robust Evaluation of Retrieval Augmented Generation
- **来源**: arXiv:2311.09476v3 (2024)
- **作者**: Jon Ander Campos, Fabio Petroni et al.
- **链接**: https://arxiv.org/abs/2311.09476
- **核心贡献**: 提出一种使用 LLM 合成偏好训练数据，并微调小型评判模型的方法，实现了高效、可复用的 RAG 端到端自动化评估，与人类判断的相关性高于 FActScore 等。

#### Benchmarking Large Language Models with Multi-Hop RAG (BGMIR)
- **来源**: arXiv:2401.06406v1 (2024)
- **工作**: 构建专门针对多跳推理的 RAG 评估集，揭示当前 LLM 在处理需要组合多文档信息时的缺陷。

## 7. 未来趋势与研究方向

1. **Agentic RAG (智能体 RAG)**: RAG 系统不再是被动检索器，而是一个能够动态规划、工具调用、自我修正的智能体。例如，使用查询分解、选择不同数据源、验证检索结果等。代表性框架有 LangGraph, LlamaIndex Agent。
   
2. **面向特定领域的深度定制**: 医疗、法律、金融等专业领域对准确性要求极高，催生了 Domain-Specific RAG。需要构建领域知识图谱、微调专用 Embedding 模型和 Reranker。

3. **成本优化与加速**: 高效检索（如乘积量化、磁盘索引）、缓存检索结果策略（GPTCache）、推测式检索等以降低 API 调用成本。

4. **多模态 RAG**: 检索对象扩展至表格、图像、音频，生成也可能包含多模态内容。如 MM-RAG。

5. **可信 RAG (Trustworthy RAG)**: 更精细的归因（Answer Attribution）、不确定性量化、防恶意注入（Indirect Prompt Injection 防御）。

#### FlashRAG: Fast and Efficient Retrieval-Augmented Generation with Real-Time Retrieval
- **来源**: arXiv:2405.13576v1 (2024)
- **作者**: 研究团队
- **链接**: https://arxiv.org/abs/2405.13576
- **核心贡献**: 通过异步检索与推测性读取的方式，将 RAG 的推理延迟降低 30-50%，适用于对实时性要求高的场景。

## 8. 总结

RAG 已从 2020 年的单一架构演变为一个繁荣的生态，涵盖了检索技术、生成控制、评估、多模态和智能体等多个子领域。其核心矛盾始终是**信息检索的覆盖率与精确度之间的平衡**，以及**外部知识注入与模型内生化能力的协调**。目前工业界的主流方案是“混合检索+重排序+精细 Prompt 编排”，而学术界仍在探索更高效的联合训练、更复杂推理以及自动化评估方法。

在选型 RAG 方案时，建议：
- **评估数据复杂度和多跳需求**，决定是否引入 GraphRAG 或迭代检索。
- **关注延迟和预算**，选择合适的检索器规模、是否使用重排序和缓存策略。
- **建立自动化评测流水线**（如基于 RAGAS），持续迭代。
