## RAG 技术深度调研报告

### 1. 引言：RAG 为何重要

**检索增强生成（Retrieval-Augmented Generation, RAG）**是一种将信息检索（Information Retrieval, IR）与大语言模型（LLM）生成能力相结合的技术范式。其核心理念是在生成回答之前，从外部知识库中检索相关文档片段，并将其作为上下文注入给 LLM，从而有效：

- **缓解幻觉**：将生成过程锚定于可溯源的证据上。
- **突破知识截止日**：允许模型访问最新或私有数据，无需微调。
- **提升知识密集型任务的可靠性**：在开放域问答、事实验证等场景中表现突出。

RAG 自 Lewis 等人 2020 年提出以来，已从简单的“检索-阅读”模式演化为一种复杂、模块化、可自适应调度的系统工程，成为 LLM 落地应用的核心架构。

---

### 2. RAG 的基本工作流程与核心组件

标准的 RAG 流程可分解为**索引（Indexing）**、**检索（Retrieval）**、**生成（Generation）**三个阶段。

#### 2.1 索引
- **文档加载与解析**：读取 PDF、HTML 等多种格式，提取纯文本。
- **文本分块**：将长文档切割为适当大小的段落（chunks），平衡语义完整性与检索粒度。
- **向量化**：通过嵌入模型将文本块映射为高维向量。
- **存储**：将向量及其元数据存入向量数据库。

#### 2.2 检索
- **查询向量化**：将用户问题编码为同一向量空间的查询向量。
- **相似度搜索**：基于余弦相似度、欧氏距离等召回 Top-K 相关文档块。常用近似最近邻（ANN）算法加速。
- **可选环节**：元数据过滤、混合检索（稀疏+稠密）、重排序（Rerank）。

#### 2.3 生成
- **上下文构建**：将检索到的文档片段与用户查询、预定义提示模板拼接。
- **LLM 生成**：调用大模型，基于该组合输入生成连贯、有据可查的回答。

**奠基论文：**
```
#### Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks
- **来源**: NeurIPS 2020, arXiv:2005.11401
- **作者**: Patrick Lewis et al.
- **链接**: https://arxiv.org/abs/2005.11401
- **核心贡献**: 首次提出 RAG 框架，将预训练的检索器（Dense Passage Retrieval）与生成器（BART）端到端联合训练，在开放式问答、事实验证等任务上达到 SOTA。
```

### 3. 核心技术栈与发展

#### 3.1 先进检索策略

早期的 RAG 主要依赖**稀疏检索**（如 BM25）或**稠密检索**（如 DPR），当前主流已演化为多种检索方式的融合。

- **混合检索**：将稀疏检索的关键词精确匹配能力与稠密检索的语义理解能力结合。典型的实践是融合 BM25 分数与向量相似度分数（如使用倒数排名融合 RRF）。
- **多向量检索**：将文档编码为多个向量（如每段、每个实体），然后在查询时取文档内最大相似度，代表模型 ColBERT。
```
#### ColBERT: Efficient and Effective Passage Search via Contextualized Late Interaction over BERT
- **来源**: SIGIR 2020, arXiv:2004.12832
- **作者**: Omar Khattab and Matei Zaharia
- **链接**: https://arxiv.org/abs/2004.12832
- **核心贡献**: 提出延迟交互（Late Interaction）范式，查询与文档的匹配通过 token 级余弦相似度求和实现，兼顾计算效率与语义匹配精确度，催生出 ColBERTv2、PLAID 等后续工作。
```

#### 3.2 分块优化

分块策略直接影响检索召回的上限。过于粗糙的固定长度分块会割裂语义，过细则丢失上下文。

- **基于文档结构**：按自然段、标题、列表等分隔符分割。
- **语义分块**：利用嵌入相似度拐点动态确定语义边界。
- **Small-to-Big 检索**：检索时使用小粒度块（如句子）以提高召回，生成时返回其所在的大粒度块（如段落）以保证上下文完整。
```
#### Recursive Character Text Splitting 与 Semantic Chunking
- **来源**: 主流框架 LangChain, LlamaIndex 的工程实践。
- **无单篇论文**：但可参考《Greg Kamradt (2024) - "Chunking Strategies for RAG"》等在线深度评测。
```

#### 3.3 嵌入模型与重排序模型

- **嵌入模型**：从 text-embedding-ada-002 到旗舰模型 text-embedding-3-large (OpenAI)，再到开源 E5、BGE、GTE 系列，普遍支持可调节维度、Matryoshka 表示学习，并针对检索进行对比预训练。
```
#### Text Embeddings by Weakly-Supervised Contrastive Pre-training
- **来源**: arXiv:2212.03533 (E5 系列基础)
- **作者**: Liang Wang et al.
- **链接**: https://arxiv.org/abs/2212.03533
- **核心贡献**: 提出 E5 嵌入模型，通过弱监督对比预训练在多种 BEIR 检索基准上取得强性能。
```

- **重排序模型**：用于对召回结果进行精确重排。基于交叉编码器的模型（如 Cohere Rerank, BGE-Reranker）能捕获查询-文档深度交互，大幅提升最终的前置精度（Precision@K）。
```
#### BGE Reranker: 基于交叉编码器的通用重排序模型
- **来源**: 可参考 BGE 论文 arXiv:2309.07597
- **作者**: Shitao Xiao et al.
- **链接**: https://arxiv.org/abs/2309.07597
- **核心贡献**: 提出 BGE 系列模型，包括高效的重排序器，通过多阶段训练获得通用能力。
```

#### 3.4 检索增强生成策略

- **简单注入**：将 Top-K 文档全部塞入上下文，不做控制。
- **提示增强**：在提示中要求模型“基于以下检索到的文档，如果无法回答就说不知道”，避免幻觉。
- **上下文压缩**：当检索文档过多、过长时，先使用 LLM 或小型摘要模型压缩冗余内容，提取与查询相关的关键信息。
- **FILCO 等方法**：训练一个过滤模型，从检索文档中抽取真正支持答案的句子，剔除噪音。

### 4. 进阶范式：从 “Retrieve-Read” 到模块化智能体

原始 RAG 是固定管道，无法适应不同复杂性的问题。新一代 RAG 引入了控制流、记忆和迭代能力。

#### 4.1 模块化 RAG（Modular RAG）
不再拘泥于线性流程，而是根据需要组合可插拔的模块（搜索、记忆、融合、重写等）。**Rewrite-Retrieve-Read** 即一例，在检索前利用 LLM 改写或分解用户查询。
```
#### Query Rewriting for Retrieval-Augmented Large Language Models
- **来源**: EMNLP 2023, arXiv:2305.14283
- **作者**: Xinbei Ma et al.
- **链接**: https://arxiv.org/abs/2305.14283
- **核心贡献**: 提出 “Rewrite-Retrieve-Read” 架构，让 LLM 先生成更有利于检索的查询，再执行检索与阅读，显著提升复杂问题检索的相关性。
```

#### 4.2 Self-RAG（自适应 RAG）
通过 LLM 在生成过程中插入特殊的 reflection token，实现**按需检索**和**自我批判**。模型会判断是否需要检索、检索到的内容是否相关、是否支持生成内容以及整体是否有用。
```
#### Self-RAG: Learning to Retrieve, Generate, and Critique through Self-Reflection
- **来源**: arXiv:2310.11511 (2023)
- **作者**: Akari Asai et al.
- **链接**: https://arxiv.org/abs/2310.11511
- **核心贡献**: 训练 LLM 自主决定何时检索，并通过批评标记对检索和生成质量进行自反馈，在各种任务上全面优于传统 RAG 和长上下文模型。
```

#### 4.3 Corrective RAG
引入检索评估器，对已检索文档打分；当相关性低于阈值时，自动触发知识精炼（如二次检索或搜索网页补充），以修正低质量召回。
```
#### Corrective Retrieval Augmented Generation
- **来源**: arXiv:2401.15884 (2024)
- **作者**: Shi-Qi Yan et al.
- **链接**: https://arxiv.org/abs/2401.15884
- **核心贡献**: 提出 CRAG，对检索结果进行置信度评估，并自动触发纠错操作（网页搜索、知识细化），用少量组件实现鲁棒的自适应生成。
```

#### 4.4 Graph RAG 与结构增强
传统 RAG 基于平面向量搜索，无法处理跨越多个文档的复杂关系。Graph RAG 利用图结构（如知识图谱、社区摘要）进行信息检索，锁定回答全局问题所需的关联实体。
```
#### From Local to Global: A Graph RAG Approach to Query-Focused Summarization
- **来源**: arXiv:2404.16130 (2024)
- **作者**: Darren Edge et al. (Microsoft Research)
- **链接**: https://arxiv.org/abs/2404.16130
- **核心贡献**: 提出 Graph RAG，先利用 LLM 从源文档构建实体图及社区摘要，再通过图遍历与摘要合成回答全局性问题，在数据集如 Podcasts 上展现优异的事实性与连贯性。
```

- 其他重要工作如 **HippoRAG**（仿人脑海马体，结合嵌入与图进行长程关联检索）、**RAPTOR**（通过递归嵌入聚类构建树状摘要索引）等，均意图突破平面检索的局限。

#### 4.5 Agentic RAG（智能体 RAG）
将检索和生成过程交给一个可调用工具、进行多步规划的自主智能体。系统不再是一次性“检索→生成”，而是迭代进行：思考→调用搜索引擎/维基/计算器→综合分析→验证。
- 代表性框架如 **LangChain Agents**、**AutoGPT**、**CrewAI** 等在 RAG 上的延伸，以及**ReAct**等推理-行动协同范式。
```
#### ReAct: Synergizing Reasoning and Acting in Language Models
- **来源**: ICLR 2023, arXiv:2210.03629
- **作者**: Shunyu Yao et al.
- **链接**: https://arxiv.org/abs/2210.03629
- **核心贡献**: 提出 ReAct 范式，交错推理痕迹与行动（如搜索），在知识密集型任务中显著减少幻觉，为 Agentic RAG 奠定基础。
```

### 5. 评估方法与基准

RAG 评估需要同时衡量**检索质量**和**生成质量**。

- **RAGAS**：目前最流行的无参考评估框架，从忠于上下文（Faithfulness）、答案相关性（Answer Relevance）、上下文相关性（Context Relevance）等维度自动化评分。
```
#### RAGAS: Automated Evaluation of Retrieval Augmented Generation
- **来源**: arXiv:2309.15217 (2023)
- **作者**: Shahul Es et al.
- **链接**: https://arxiv.org/abs/2309.15217
- **核心贡献**: 提出无参考的 RAG 评估指标集，通过链式提示计算忠实度、相关性等，无需人工标注，极大推动 RAG 系统迭代效率。
```

- **RGB**：该基准测试将分段前后信息分散在多个文档中，要求模型推理并整合，专门评估 RAG 的信息整合能力。
*参考：Chen et al., "Benchmarking Large Language Models in Retrieval-Augmented Generation", AAAI 2024, arXiv:2309.01431*

- **ARES**：通过合成数据训练轻量级预测模型，自动化评分上下文相关、答案忠实等，相比基于 LLM 的裁判更稳定、成本更低。
*参考：Saad-Falcon et al., "ARES: An Automated Evaluation Framework for Retrieval-Augmented Generation Systems", arXiv:2311.09476*

- **TruLens**（TruEra 开发的评估工具）提供 RAG 三元素（答案真实性、上下文相关性、确定性）的反馈功能。

### 7. 挑战与未来方向

1. **长输入与长上下文的融合**：Gemini 1.5 Pro、GPT-4-Turbo 等模型的上下文窗口可达百万 token，“直接将整个文本投喂”的模式开始挑战传统 RAG 的必要性。但成本与首 token 延迟仍促使研究者探索**长上下文 RAG**（LoCo）等融合方案，按需分段检索最佳块灌入长窗。
```
#### Retrieval Augmented Generation or Long-Context LLMs? A Comprehensive Study and Hybrid Approach
- **来源**: arXiv:2407.16833 (2024)
- **作者**: Zhuowan Li et al.
- **链接**: https://arxiv.org/abs/2407.16833
- **核心贡献**: 系统性比较 RAG 与长上下文 LLM，证明两者各有优势，并提出 SELF-ROUTE 混合策略，路由简单查询至长上下文、复杂查询至 RAG，平衡性能与成本。
```

2. **多模态 RAG**：从文本扩展到图像、表格、音频等多种数据源，如何统一索引、跨模态对齐与生成是重要方向。
```
#### MM-RAG: Multi-Modal Retrieval-Augmented Generation
- **来源**: arXiv:2310.16273 (相关工作)
- **核心趋势**：整合视觉编码器与嵌入模型，实现对多模态文档的检索与增强生成。
```

3. **可信与可解释性**：检索出的文档可能是过时、片面甚至被投毒的。需要细颗粒度的引用溯源、检索事实核查以及来源权威性评估机制。

4. **缓存与成本效率**：通过语义缓存（如 GPTCache）保存相似查询的 API 结果，降低延迟与费用。

5. **Task-specific 微调与 RLHF**：针对特定领域微调嵌入模型、重排序模型甚至生成模型，提升端到端的一致性。DSG（DeepSeek 等公司开始实践）通过强化学习直接将“检索准确性”纳入奖励函数进行优化，将是下一代 RAG 令人振奋的方向。

---

本调研梳理了 RAG 从基础范式到最新前沿的完整技术脉络。当前 RAG 已经超越了简单的“检索+提示”，发展为包含自适应路由、图结构知识索引、多步智能体推理的复杂系统。随着长上下文模型和多模态技术的演变，RAG 将继续在**成本、延迟与效果之间寻找最优平衡点**，并成为构建可信、知识驱动 AI 的核心基础设施。
