# RAG 技术深度调研报告

## 1. 引言与背景

**RAG（Retrieval-Augmented Generation，检索增强生成）** 是一种将信息检索系统与大语言模型（LLM）结合的技术范式。其核心思路是：在生成回答之前，先从外部知识库中检索相关文档片段，然后将这些检索到的内容作为上下文提供给 LLM，以生成更准确、知识更新、可溯源且减少幻觉的文本。

大语言模型虽然在许多任务上表现出色，但面临知识截止日期固定、容易产生幻觉、缺乏领域专业知识等问题。RAG 通过引入外部知识，使模型能够“边查边答”，极大地扩展了 LLM 的应用边界，尤其是在知识密集型任务、开放域问答、企业知识库、对话系统等场景中。

## 2. 基础架构与核心论文

典型的 RAG 系统包含三个核心模块：**检索器（Retriever）**、**生成器（Generator）** 和**信息融合机制**。根据训练方式不同，可分为“冻结检索器+微调生成器”、“联合微调”和“端到端可微分检索”等范式。

### 2.1 原始 RAG 框架

#### 论文：Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks
- **来源**: arXiv:2005.11401 (2020)
- **作者**: Patrick Lewis, Ethan Perez, Aleksandra Piktus, et al. (Facebook AI)
- **链接**: https://arxiv.org/abs/2005.11401
- **核心贡献**: 首次提出 RAG 概念，将预训练的参数记忆（生成模型）与非参数记忆（基于维基百科的密集向量检索）结合。模型使用 DPR 作为检索器，BART 作为生成器，在生成时从检索到的文档中抽取证据。论文探索了两种集成方式：**RAG-Sequence**（同一检索集生成完整序列）和**RAG-Token**（每个 token 可依赖不同检索文档）。在开放域问答任务上显著优于纯参数模型。

### 2.2 密集检索基础：DPR

#### 论文：Dense Passage Retrieval for Open-Domain Question Answering
- **来源**: arXiv:2004.04906 (2020)
- **作者**: Vladimir Karpukhin, Barlas Oğuz, Sewon Min, et al.
- **链接**: https://arxiv.org/abs/2004.04906
- **核心贡献**: 提出使用双编码器架构，将问题和段落分别编码为稠密向量，通过近似最近邻搜索实现高效检索。训练过程中使用同一 batch 内的其他段落作为负例，极大提升了开放域检索的精度，成为后续 RAG 系统的标准检索组件。

### 2.3 端到端可微分检索：REALM

#### 论文：REALM: Retrieval-Augmented Language Model Pre-Training
- **来源**: arXiv:2002.08909 (2020)
- **作者**: Kelvin Guu, Kenton Lee, Zora Tung, et al.
- **链接**: https://arxiv.org/abs/2002.08909
- **核心贡献**: 在预训练阶段即引入检索，通过异步更新检索索引，实现检索器和编码器的联合端到端学习。使用掩码语言建模目标，让模型学会在知识语料库中寻找所需信息。该方法首次展示了检索增强预训练的有效性。

### 2.4 融合解码器：FiD (Fusion-in-Decoder)

#### 论文：Leveraging Passage Retrieval with Generative Models for Open Domain Question Answering
- **来源**: arXiv:2007.01282 (2020)
- **作者**: Gautier Izacard, Edouard Grave
- **链接**: https://arxiv.org/abs/2007.01282
- **核心贡献**: 提出将检索到的多个文档分别编码，然后在生成器的解码器部分进行融合。具体做法是将每个检索片段与问题拼接后独立编码，所有编码向量展平后喂入解码器交叉注意力。这种设计允许模型在生成每个 token 时利用所有检索信息，在 TriviaQA 和 Natural Questions 上达到当时最佳性能。

## 3. 检索技术深度解析

| 检索类型 | 代表方法 | 查询速度 | 召回精度 | 可解释性 | 多语言支持 | 典型使用场景 |
|----------|---------|----------|----------|----------|------------|-------------|
| 稀疏检索 (BM25) | Elasticsearch 内置 BM25 | 极快 | 关键词匹配强 | 高(WORD匹配) | 支持但需分词 | 法律、代码、精确匹配 |
| 密集检索 | DPR, Contriever, GTR | 快 (需索引) | 语义泛化强 | 低(向量相似) | 通过多语言模型支持 | 开放域问答、模糊需求 |
| 混合检索 | BM25 + DPR 线性组合 | 中 | 高 | 中 | 可组合 | 企业搜索、通用 RAG |
| 多向量检索 | ColBERT, SPLADE | 慢 (存储大) | 极高 | 中 | 基于模型 | 高精度学术搜索 |

### 3.1 稀疏检索

传统的 BM25 算法基于词频-逆文档频率（TF-IDF）的改进，计算查询和文档的词汇重叠度。优点是不需要训练，运行高效，对确切术语匹配极佳。缺点是无法处理同义词、语义近似，受限于词汇鸿沟。

### 3.2 密集检索

密集检索利用预训练语言模型将文本映射到固定维度的向量空间，通过向量相似性衡量语义关联。

- **DPR** (arXiv:2004.04906): 独立训练问题编码器和段落编码器，使用对比学习。训练数据通常是问答对配相关段落。
- **Contriever** (arXiv:2112.09118): 无监督对比学习，利用文本片段之间的互信息，不需要标注数据。
- **GTR** (arXiv:2112.07899): 大型双编码器，采用 T5 初始化，以零样本方式泛化到多个检索任务。

### 3.3 混合检索与多向量

混合检索结合稀疏和密集分数的加权和（如 `score = α·BM25 + (1-α)·DPR`），在实践中被广泛验证能够提升召回率和鲁棒性。

多向量检索（如 **ColBERT** arXiv:2004.12832）对每个 token 产生一个向量，进行延迟交互（MaxSim），在列级别保留细粒度匹配。**SPLADE** (arXiv:2107.05720) 通过扩展查询词汇来增强稀疏向量。

## 4. 检索增强的生成与集成方式

生成器与检索器的集成策略决定了信息利用的效率。常见方法包括：

| 集成方式 | 核心思想 | 代表工作 | 优点 | 缺点 |
|----------|---------|---------|------|------|
| 前置上下文拼接 | 检索片段直接拼入 Prompt | 原始 RAG, LangChain 范式 | 实现简单，无需改动模型 | 受上下文长度限制，噪声干扰大 |
| 交叉注意力融合 | 生成器的交叉注意力层访问所有检索结果 | FiD (arXiv:2007.01282) | 全局利用多文档信息 | 编码计算量大，延迟高 |
| 检索令牌增强 | 引入特殊检索令牌，模型自行决定检索时机 | RETRO (arXiv:2112.04426) | 动态、检索粒度细 | 需要修改模型架构和预训练 |
| 工具调用式检索 | 生成过程调用检索 API，如同外部工具 | Toolformer, ReAct | 灵活，与 LLM 解耦 | 调用决策依赖 prompt engineering |

- **RETRO** (arXiv:2112.04426): 在自回归生成过程中，每生成一个 chunk 时检索相似 chunk，并通过块状交叉注意力融合。需要定制预训练，但具备极强的知识扩展性。
- **ReAct** (arXiv:2210.03629): 将推理和行动交织，允许模型在生成过程中生成“行动指令”触发检索，读取结果后继续生成，为更复杂的 Agentic RAG 提供了基础。

## 5. 高级 RAG 策略

为了克服基础 RAG 的局限（如检索一次、不分质量、无法纠正错误等），一系列高级策略被提出。

### 5.1 自反思检索与纠错

#### Self-RAG: Learning to Retrieve, Generate, and Critique through Self-Reflection
- **来源**: arXiv:2310.11511 (2023)
- **作者**: Akari Asai, Zeqiu Wu, Yizhong Wang, et al.
- **链接**: https://arxiv.org/abs/2310.11511
- **核心贡献**: 训练 LLM 在生成时按需检索，并对检索到的段落和自身生成内容进行事实性、相关性反思。模型生成特殊反射令牌（Retrieve, IsRel, IsSup, IsUse）来评判。实现了检索时机、文档选择、生成质量控制的统一。

#### CRAG: Corrective Retrieval Augmented Generation
- **来源**: arXiv:2401.15884 (2024)
- **作者**: Shi-Qi Yan, Jia-Chen Gu, Yun Zhu, et al.
- **链接**: https://arxiv.org/abs/2401.15884
- **核心贡献**: 引入检索评估器，对检索文档的质量评分。若置信度低，则触发网络搜索进行补充或纠正。然后经过“分解-再重组”处理，过滤无关信息，最终生成。该方法显著提升了生成的事实准确性。

#### FLARE: Active Retrieval Augmented Generation
- **来源**: arXiv:2305.06983 (2023)
- **作者**: Zhengbao Jiang, Frank F. Xu, Luyu Gao, et al.
- **链接**: https://arxiv.org/abs/2305.06983
- **核心贡献**: 在生成过程中监测生成句子的置信度，当 LLM 不确定时，主动使用生成的置信度较低的关键词句进行检索，并重新生成。实现了前瞻性、主动式的动态检索。

### 5.2 面向领域适应与微调

#### RAFT: Adapting Language Model to Domain Specific RAG
- **来源**: arXiv:2403.10131 (2024)
- **作者**: Tianjun Zhang, Shishir G. Patil, Naman Jain, et al.
- **链接**: https://arxiv.org/abs/2403.10131
- **核心贡献**: 提出在微调阶段，将问题、检索文档（部分相关，部分噪声）和思维链风格答案组合作为训练数据。要求模型学会从混合文档中提取有用信息并引用。使得模型在特定领域的“开卷考试”中表现优异。

### 5.3 图谱增强 RAG

#### GraphRAG: Unsupervised Graph-based Retrieval Augmented Generation
- **来源**: arXiv:2404.16130 (2024) (Microsoft Research 的技术报告)
- **作者**: Darren Edge, Ha Trinh, Newman Cheng, et al.
- **链接**: https://arxiv.org/abs/2404.16130
- **核心贡献**: 利用 LLM 从源文档中提取实体和关系，构建知识图谱，并生成社区摘要。在问答时，先基于图进行局部检索（如邻居实体、社区摘要），将结构化信息提供给 LLM。相比传统基准 RAG，在处理全局查询（如总结整个数据集）时表现大幅提升。

### 5.4 迭代与递归检索

**IRCoT** (arXiv:2212.10509) 提出将检索过程与思维链推理步骤交织，每步推理后检索新信息。**Tree-RAG** 和 **RAPTOR** (arXiv:2401.18059) 递归地构建文档树结构，从叶节点到高层摘要进行多粒度检索。

## 6. 评估指标与基准

RAG 系统的评估涉及检索质量和生成质量两个维度。

| 评估维度 | 常用指标 | 说明 |
|----------|---------|------|
| 检索质量 | Recall@k, Precision@k, MRR, NDCG@k | k 通常取 5, 10, 20 |
| 答案质量 | Exact Match (EM), F1 | 常用于抽取式问答 |
| 事实一致性 | Faithfulness, Hallucination Rate | 使用 NLI 模型或专门事实性评估工具 (如 AlignScore) |
| 归因能力 | Citation Recall, Citation Precision | 生成文本中声明是否被文档支持 |
| 综合能力 | RAGAS, ARES, TruLens | 专为 RAG 设计的评估框架 |

### 重要评估框架

- **RAGAS** (arXiv:2309.15217): 提供无参考的 RAG 评估指标，包括上下文相关性、答案忠实度、上下文召回率等。已成为开源社区广泛采用的工具。
- **KILT** (arXiv:2009.02252): 统一的知识密集型语言任务基准，涵盖事实核查、问答、实体链接等，要求同时提供来源证据。
- **RGB** (arXiv:2307.03172): 专门评估 RAG 的中文基准，包含噪声鲁棒性、信息整合和反事实鲁棒性任务。

## 7. 挑战与未来方向

### 7.1 当前挑战

1. **检索质量与噪声**: 检索到的无关或误导性文档会降低生成质量。即便使用纠错机制，低质量检索仍可能导致错误累积。
2. **长上下文与多文档整合**: 虽然 LLM 上下文窗口不断扩大，但大量检索文档的线性拼接会稀释关键信息，引发“迷失在中间”现象。
3. **事实归因与幻觉**: 生成内容可能看似合理但未被文档支持，或错误引用。可靠的事实核查和句子级归因仍是难题。
4. **延迟与效率**: 多次检索、交叉注意力、重排序等操作显著增加首 token 延迟。
5. **检索时机的决策**: 何时检索、检索多少次，需要基于不确定性估计的精确决策机制。

### 7.2 前沿趋势

- **Agentic RAG**: 自主规划、多步推理、工具调用的代理式 RAG 系统，能处理复杂多源信息查询。
- **多模态 RAG**: 将文本检索扩展至图像、表格、音频等多种模态，实现跨模态知识合成。
- **可微分搜索索引**: 将检索和生成完全统一到端到端可训练的模型架构中（如 DSI, NCI）。
- **隐私保护与本地 RAG**: 结合边缘计算和本地向量库（如 LlamaIndex, privateGPT）的离线 RAG。
- **自改进与自我纠错**: 模型学习评价和修正自己的检索与生成，形成闭环自我进化（如 Self-RAG, CRAG 的进一步发展）。

## 9. 总结

RAG 技术已经从简单的“检索-拼接-生成”模式发展为具备动态决策、自反思、多步迭代以及结构化知识融合的智能增强框架。关键的技术路径包括：从双编码器到融合解码器的架构升级、从静态检索到主动检索的范式转变、以及从文本块到知识图谱的多维知识表示。面向未来，Agentic RAG、多模态融合和可信赖的事实归因将引领该领域的进一步突破。
