# RAG（检索增强生成）技术深度调研报告

## 1. 什么是 RAG：背景与动机

### 1.1 定义
检索增强生成（Retrieval-Augmented Generation, RAG）是一种将**信息检索**与**大语言模型（LLM）生成**相结合的框架，旨在通过引入外部知识来缓解 LLM 的幻觉问题、知识截止问题以及提高生成事实的准确性（Lewis et al., 2020）。

### 1.2 核心问题
- **LLM 的局限**：参数化知识容易过时，无法覆盖长尾事实，且容易生成非事实内容。
- **微调的高昂成本**：对模型进行全量微调以更新知识成本高、周期长。

### 1.3 RAG 工作流程
1. **Query**：用户提出问题。
2. **Retrieval**：检索系统从外部知识库（文档、数据库等）中查找相关文本片段。
3. **Augmentation**：将检索到的文本作为上下文拼接到 Prompt 中。
4. **Generation**：LLM 基于增强后的 Prompt 生成回答。

## 2. 基础范式与经典架构

### 2.1 经典 RAG (Naive RAG)
#### 原始论文
##### Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks
- **来源**: NeurIPS 2020
- **arXiv**: 2005.11401
- **作者**: Patrick Lewis, Ethan Perez, et al.
- **链接**: [https://arxiv.org/abs/2005.11401](https://arxiv.org/abs/2005.11401)
- **核心贡献**: 提出了 RAG 框架，将预训练的检索器（Dense Passage Retrieval，DPR）与预训练的序列到序列模型（BART）结合，端到端训练。包含两种形式：RAG-Sequence（对整个序列使用同一个检索文档）和 RAG-Token（对每个 Token 使用不同检索文档）。

**架构组件**：
- **查询编码器**：BERT-base 编码问题。
- **文档索引**：基于 DPR 的密集向量索引。
- **检索器**：最大内积搜索（MIPS）。
- **生成器**：BART-large。

### 2.2 Dense Passage Retrieval (DPR)
经典 RAG 的检索基础。
##### Dense Passage Retrieval for Open-Domain Question Answering
- **来源**: EMNLP 2020
- **arXiv**: 2004.04906
- **作者**: Vladimir Karpukhin, Barlas Oğuz, et al.
- **链接**: [https://arxiv.org/abs/2004.04906](https://arxiv.org/abs/2004.04906)
- **核心贡献**: 证明密集向量检索在开放域问答上显著优于稀疏检索（BM25），使用双编码器架构并基于对比学习训练。

## 4. 核心组件与技术优化

### 4.1 索引优化（Indexing）
文档切分为块，转化为向量存储。

- **分块策略**：固定长度、滑动窗口、基于语义的分块。
- **细粒度索引**：
  - **Sentence Window Retrieval**：检索句子，但扩展上下文窗口送给 LLM。
  - **Parent Document Retriever**（LangChain）：检索小子块，返回其所属的完整父文档。
- **多粒度索引**：同时建立文档级、段落级、句子级索引。
- **查询感知索引**：为每个文档块生成假设性问题，用问题-答案对建立索引，在检索时匹配用户问题和假设问题。

#### 代表性方法 - HyDE
##### Precise Zero-Shot Dense Retrieval without Relevance Labels
- **来源**: arXiv 2022
- **arXiv**: 2212.10496
- **作者**: Luyu Gao, Xueguang Ma, et al.
- **链接**: [https://arxiv.org/abs/2212.10496](https://arxiv.org/abs/2212.10496)
- **核心贡献**: 提出 Hypothetical Document Embeddings，先让 LLM 根据问题生成假想文档，再用该假想文档的向量去检索真实文档，减轻查询和文档的语义 gap。

### 4.2 检索优化（Retrieval）
- **混合检索**：结合稀疏检索（BM25）和密集检索（向量），互补关键词和语义匹配。
- **重排序（Re-ranking）**：使用交叉编码器对初检结果重新排序。
  - **Cohere Rerank**、**BGE Re-Ranker**、**ColBERT**。

#### ColBERT 相关
##### ColBERT: Efficient and Effective Passage Search via Contextualized Late Interaction over BERT
- **来源**: SIGIR 2020
- **arXiv**: 2004.12832
- **作者**: Omar Khattab, Matei Zaharia
- **链接**: [https://arxiv.org/abs/2004.12832](https://arxiv.org/abs/2004.12832)
- **核心贡献**: 延迟交互机制，保留 Token 级向量，在检索时效和效果间平衡，可作为重排序。

### 4.3 生成增强（Augmentation & Generation）
- **上下文编排**：如何把检索到的多个片段组织成 Prompt。
- **注意力增强**：让 LLM 的注意力额外关注检索文档。
- **迭代检索**：
  - **Self-RAG**：模型在生成过程中自适应检索。
  - **IRCoT**：将 CoT 推理与检索交替进行。

#### Self-RAG
##### Self-RAG: Learning to Retrieve, Generate, and Critique through Self-Reflection
- **来源**: arXiv 2023
- **arXiv**: 2310.11511
- **作者**: Akari Asai, Zeqiu Wu, et al.
- **链接**: [https://arxiv.org/abs/2310.11511](https://arxiv.org/abs/2310.11511)
- **核心贡献**: 训练 LM 在生成过程中按需检索，并生成反思 Token 来评判检索段落相关性和生成质量，实现选择性检索与自我修正。

#### IRCoT
##### Interleaving Retrieval with Chain-of-Thought Reasoning for Knowledge-Intensive Multi-Step Questions
- **来源**: ACL 2023
- **arXiv**: 2212.10509
- **作者**: Harsh Trivedi, Niranjan Balasubramanian, et al.
- **链接**: [https://arxiv.org/abs/2212.10509](https://arxiv.org/abs/2212.10509)
- **核心贡献**: 交替进行思维链推理和检索，每一步思维引导下一步检索，大幅提升多步推理问答性能。

## 5. Graph RAG：从文本到知识图谱

当文档间逻辑关系复杂，全局理解需求高时，基于图谱的 RAG 优越。

#### 微软 GraphRAG
##### From Local to Global: A Graph RAG Approach to Query-Focused Summarization
- **来源**: arXiv 2024
- **arXiv**: 2404.16130
- **作者**: Darren Edge, Ha Trinh, et al. (Microsoft)
- **链接**: [https://arxiv.org/abs/2404.16130](https://arxiv.org/abs/2404.16130)
- **核心贡献**: 利用 LLM 从原始文本构建实体与关系知识图谱，再通过社区检测算法将图分解为社区，对每个社区生成摘要。回答问题时可组合社区摘要形成全局语境，显著提升长文档、大型语料库上的全局理解与总结能力。

**工作流程**：
1. 源文档 → 文本块。
2. 每个块提取实体、关系、Covariates（声明）。
3. 构建实体-关系网络图。
4. 检测层级社区。
5. 为每个社区生成自下而上的报告摘要。
6. 根据问题聚合社区摘要生成最终回答。

**Graph RAG 其他形式**：
- **基于已有 KG 的 RAG**：直接使用 Wikidata、企业知识图谱作为检索源，将问题转为查询语句（SPARQL/Cypher），返回三元组或子图。
- **HippoRAG**（OSU/Stanford, 2024）：受人大脑海马体记忆索引理论启发，结合 LLM 和知识图谱实现更仿生的长期记忆检索。

## 6. Agentic RAG：智能体驱动的检索

Agentic RAG 赋予 LLM 自主规划、工具使用和多次迭代的能力。

### 主要特征
- **动态、多步检索策略**：代理根据已获取信息决定是否需要检索更多、检索什么。
- **工具组合**：除了向量搜索，还可调用 Web 搜索、计算器、数据库查询、APIs。
- **记忆与状态**：维护多轮交互的历史和检索历史，避免重复。

#### Adaptive RAG
##### Adaptive-RAG: Learning to Adapt Retrieval-Augmented Large Language Models through Question Complexity
- **来源**: COLING 2024
- **arXiv**: 2403.14403
- **作者**: Soyeong Jeong, Jinheon Baek, et al.
- **链接**: [https://arxiv.org/abs/2403.14403](https://arxiv.org/abs/2403.14403)
- **核心贡献**: 根据不同查询的复杂度动态选择策略，从无检索单步到多步检索，使用分类器判断复杂度级别。

#### Corrective RAG (CRAG)
##### Corrective Retrieval Augmented Generation
- **来源**: arXiv 2024
- **arXiv**: 2401.15884
- **作者**: Shi-Qi Yan, Jia-Chen Gu, et al.
- **链接**: [https://arxiv.org/abs/2401.15884](https://arxiv.org/abs/2401.15884)
- **核心贡献**: 引入检索评估器，对检索文档打分，若置信度低则自动调用 Web 搜索进行校正，之后再进行知识精炼与生成。

## 7. 多模态 RAG

扩展现有 RAG 至图像、表格、音频等多模态场景。

#### Multi-Modal RAG 典型实现
- **检索对象**：图片 CLIP 嵌入、图表的文本描述、表格的 Markdown 格式。
- **Augmentation**：将检索到的多模态元素（图片 Base64、表格文本）与文本上下文混合输入多模态 LLM（如 GPT-4V、Gemini）。

#### MuRAG
##### MuRAG: Multimodal Retrieval-Augmented Generator for Open Question Answering over Images and Text
- **来源**: EMNLP 2022
- **arXiv**: 2210.02928
- **作者**: Wenhu Chen, Hexiang Hu, et al.
- **链接**: [https://arxiv.org/abs/2210.02928](https://arxiv.org/abs/2210.02928)
- **核心贡献**: 在多模态多文档问答中，联合检索文本和图像，并通过预训练多模态编码器融合，再生成答案。

## 8. 评估与基准

### 8.1 关键指标
- **忠实度（Faithfulness）**：生成内容是否完全基于检索上下文。
- **答案相关性（Answer Relevance）**：答案与问题的匹配度。
- **上下文召回率（Context Recall）**：检索结果对真实参考答案的覆盖度。
- **上下文精确度（Context Precision）**：检索结果中相关文档的排序质量。

### 8.2 评价框架与基准
#### RAGAS (RAG Assessment)
##### RAGAS: Automated Evaluation of Retrieval Augmented Generation
- **来源**: EACL 2024
- **arXiv**: 2309.15217
- **作者**: Shahul Es, Jithin James, et al.
- **链接**: [https://arxiv.org/abs/2309.15217](https://arxiv.org/abs/2309.15217)
- **核心贡献**: 提出一套无需人工标注、基于 LLM 的自动化评价指标，配合 RAG 评估数据集。

#### RGB Benchmark
##### Benchmarking Large Language Models in Retrieval-Augmented Generation
- **来源**: AAAI 2024
- **arXiv**: 2309.01431
- **作者**: Jiawei Chen, Hongyu Lin, et al.
- **链接**: [https://arxiv.org/abs/2309.01431](https://arxiv.org/abs/2309.01431)
- **核心贡献**: 设计了评测中文和英文 LLM 在 RAG 下的噪声鲁棒性、弃权能力、信息整合能力、反事实鲁棒性等四个维度的基准。

## 9. 关键挑战与未来方向

### 9.1 挑战
1. **检索准确性与上下文优化**：检索失败或返回冗余内容会损害生成质量。
2. **长上下文处理**：如何将大量检索结果高效输入 LLM（当前模型上下文窗口扩大，但计算成本增加且可能丢失中间位置信息）。
3. **鲁棒性与反事实**：抵御检索注入有害/虚假信息的能力。
4. **延迟成本**：检索、重排、生成三步走带来额外延迟。
5. **领域适应**：在医疗、法律、科学等领域需要专用嵌入模型和结构化知识。

### 9.2 未来方向
- **更紧密的联合训练**：不是简单拼接，而是让 LLM 学会如何利用检索信息。
- **个性化与记忆**：长期用户记忆和持续学习。
- **多智能体协作 RAG**：多个 Agent 分别负责检索、核查、生成。
- **高效的在线检索**：类 ChatGPT “搜索” 模式，实时网络信息整合。
- **结构化知识增强**：将表格、数据库、知识图谱深度融合。

## 11. 趋势总结

| 趋势方向 | 说明 | 代表工作 |
|----------|------|----------|
| **Naive → Modular** | 从固定流程到可插拔的模块化系统 | Modular RAG (Survey 2312.10997) |
| **静态 → 自适应** | 根据查询类型与反馈动态调整检索策略 | Self-RAG, Adaptive-RAG, CRAG |
| **文本 → 多模态** | 同时检索图像、表数据、代码等多源信息 | MuRAG, Gemini multi-modal RAG |
| **单次 → 多步/推理** | 检索需配合思维链，完成多跳推理 | IRCoT, Self-RAG |
| **局部 → 全局** | 从局部文本匹配到图形结构全局理解 | Microsoft GraphRAG, HippoRAG |
| **工具调用** | RAG 系统具备 API 调用、数据库查询等能力 | Toolformer, Agentic RAG |
| **离线评估 → 在线生产监控** | 关注真实场景中的延迟、成本、反馈闭环 | RAGAS, ARES 等评估框架 |
