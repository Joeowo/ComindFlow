# RAG（检索增强生成）技术深度调研报告

## 1. 引言：RAG 的背景与动机

大语言模型（LLM）在大量文本上预训练后，内部存储了丰富的“参数化知识”。然而，这类知识存在**更新困难、幻觉高频、可解释性弱、且难以捕捉长尾事实**等局限。检索增强生成（Retrieval-Augmented Generation，RAG）通过引入外部知识库，将LLM与检索系统结合，使模型在生成回答时能够动态查询并融合最新、相关的非参数化知识，从而显著提升事实准确性与领域适应性。

RAG 的核心工作是 Lewis 等人于 2020 年提出的，其将预训练的检索器（Bi-encoder）与序列到序列生成器联合建模，在知识密集型任务上获得了优异表现。此后，RAG 演变成一个快速迭代的研究领域，从简单的“检索-阅读”范式发展为包含**查询增强、多步推理、自省式生成、模块化编排、多模态和图谱增强**等丰富技术栈的体系。

## 2. RAG 的核心范式与架构演化

依据检索/生成之间的耦合方式和处理流程，可以将RAG体系划分为三个阶段：**朴素RAG（Naive RAG）→ 高级RAG（Advanced RAG）→ 模块化RAG（Modular RAG）**。

### 2.1 朴素 RAG（Naive RAG）

**架构流程**：索引 → 检索 → 生成。
- **索引阶段**：将文档库切分、向量化并存入向量数据库。
- **检索阶段**：用户查询被编码为向量，通过相似度匹配（如余弦相似度）返回 Top-K 文档块。
- **生成阶段**：将查询与检索到的文本拼接为提示，输入LLM生成最终回答。

**代表性论文**：
#### Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks
- **来源**: arXiv:2005.11401 (NeurIPS 2020)
- **作者**: Patrick Lewis et al.
- **链接**: https://arxiv.org/abs/2005.11401
- **核心贡献**: 首次提出 RAG 架构，联合训练检索器（DPR）和生成器（BART），展示了开放域问答任务的重大提升。

**朴素 RAG 的局限性**：
- 一次检索固定，难以应对多跳、模糊或需细粒度信息的问题。
- 对检索结果不加甄别，容易引入噪声和中段信息丢失（Lost in the Middle）。
- 生成部分仅为“读取”，缺乏对检索信息的核实和反思能力。

### 2.2 高级 RAG（Advanced RAG）

为克服朴素RAG的缺陷，研究人员提出了“重排序”、“查询重写”、“混合检索”、“上下文压缩”等增强技术，在检索前、中、后阶段优化信息质量。该方法仍保持“检索 → 生成”的线性流程，但每个环节均深度强化。

**代表性增强技术**：
- **查询改写与扩展**：如 Query2doc、HyDE、Step-Back Prompting 等。
- **多策略检索融合**：BM25（稀疏）+ 稠密向量 + 多路召回。
- **重排序（Rerank）**：利用交叉编码器对召回文档进行语义重排，缓解“Lost in the Middle”效应。

#### Lost in the Middle: How Language Models Use Long Contexts
- **来源**: arXiv:2307.03172
- **作者**: Nelson F. Liu et al.
- **链接**: https://arxiv.org/abs/2307.03172
- **核心贡献**: 证明LLM在长上下文处理中存在U型位置偏见，有用信息在中间位置时性能显著下降，这直接推动重排序与上下文压缩的需求。

### 2.3 模块化 RAG（Modular RAG）

模块化 RAG 打破线性管道，允许将检索、推理、记忆、工具调用等模块**按需组合**，形成更复杂的决策流。典型实现包括路由（Router）、调度（Scheduler）、迭代检索与 Agent 式自主决策。

**特点**：
- 支持**动态检索频率**：部分步骤检索，部分直接利用生成器的内部知识。
- 可集成多种工具（搜索引擎、计算器、代码解释器）。
- 支持**多轮推理循环**（如 IRCoT、Self-ASK）。

#### Interleaving Retrieval with Chain-of-Thought Reasoning for Knowledge-Intensive Multi-Step Questions
- **来源**: arXiv:2212.10509 (ACL 2023)
- **作者**: Harsh Trivedi et al.
- **链接**: https://arxiv.org/abs/2212.10509
- **核心贡献**: 提出 IRCoT，在思维链推理的每一步交错进行检索，极大提升多跳问答的准确率。

## 3. 关键技术精细解读

### 3.1 查询增强：从原始查询到精准语义

单一用户查询往往不够精确，研发多种查询增强方法：

| 方法                     | 核心思路                                         | 代表性论文                                                                 |
|------------------------|-------------------------------------------------|------------------------------------------------------------------------|
| HyDE (Hypothetical Document Embeddings) | 先生成假设性文档，再以其向量检索真实文档，缩小语义差异 | Gao et al., 2022, arXiv:2212.10496 |
| Step-Back Prompting    | 通过抽象生成高层“退一步”问题，辅助检索更关键信息         | Zheng et al., 2023, arXiv:2310.06117 |
| Query Rewriting (LLM)  | 利用LLM对原始查询进行分解、消歧、时间绑定等改写         | Ma et al., 2023, arXiv:2305.14625   |
| FLARE (Forward-Looking Active REtrieval) | 在生成过程中根据置信度动态决定何时检索，并利用下一句预测内容作为查询 | Jiang et al., 2023, arXiv:2305.05065 |

### 3.2 检索策略：稀疏、稠密与混合

- **稀疏检索**：BM25 依靠词频，精准匹配关键词，对实体和稀有词友好。
- **稠密检索**：基于预训练语言模型（如 DPR、Contriever），语义泛化强。
- **混合检索**：向量空间与词袋空间结果通过线性加权或学习式融合，通常在无标注下即能提升召回率。

**常用稠密检索模型**：
- DPR (2020)  
- Contriever (Izacard et al., 2021, arXiv:2112.09118)  
- E5 (Wang et al., 2022, arXiv:2212.03533)  
- BGE (BAAI)

### 3.3 检索后处理：重排序与上下文压缩

重排序阶段通常将粗筛后的 Top-K 通过强语义模型（如 Cross-encoder、基于 LLM 的评分配置）重排，仅保留最相关的数条文档，以适配生成窗口。

**典型重排序方案**：
- **基于 Cross-encoder 的重排序**：如 BERT-Reranker 系列。
- **LLM 作为排序器**：如 RankVicuna 或 GPT-4 打分。
- **压缩**：自动剔除冗余片段，仅保留生成所需的信息跨度（LongLLMLingua, RECOMP）。

#### RECOMP: Improving Retrieval-Augmented LMs with Compression and Selective Augmentation
- **来源**: arXiv:2310.04408
- **作者**: Fangyuan Xu et al.
- **链接**: https://arxiv.org/abs/2310.04408

### 3.4 面向生成的增强技术：如何用好检索信息

- **REPLUG**：将检索文档置于黑盒语言模型两侧，计算似然，通过简单加权集成多文档预测，避免微调。
- **Self-RAG**：训练模型产生特殊反思标记（Retrieve, IsRel, IsSup, IsUse），控制何时检索并评判内容，形成自我反馈闭环。
- **RAFT**：在微调时混合有答案/无答案文档，并训练模型引用文档的精确片段，用于领域适配。

#### REPLUG: Retrieval-Augmented Black-Box Language Models
- **来源**: arXiv:2301.12652

- **链接**: https://arxiv.org/abs/2301.12652

## 4. RAG 训练范式与微调

根据检索器与生成器的耦合程度，主要分为三类：

1. **独立训练**：检索器（DPR）和生成器（BART/T5）分别预训练，典型为朴素 RAG。
2. **顺次微调/联合微调**：固定一个模块，微调另一个或使用桥接适配器联合优化。  
   ・**REALM**：在语言模型预训练阶段就加入检索知识，端到端更新检索器和知识增强的MLM。  
   - 论文：Kelvin Guu et al., 2020, arXiv:2002.08909
   ・**Atlas**：少量样本学习时将检索编码器和生成器联合训练，利用共享的硬负样本提升检索。  
   - 论文：Gautier Izacard et al., 2022, arXiv:2208.03299
3. **基于强化学习的工具使用**：通过与环境交互的奖励信号来学习何时及如何检索。  
   ・**Toolformer**：将API调用视为 token，自监督训练，使模型自主使用检索工具 (Schick et al., 2023, arXiv:2302.04761)。  
   ・**ReAct**：交替产生推理轨迹和行动，使用维基百科等环境交互 (Yao et al., 2022, arXiv:2210.03629)。

**代表性论文列表**：
- REALM: Retrieval-Augmented Language Model Pre-Training – arXiv:2002.08909
- Atlas: Few-shot Learning with Retrieval Augmented Models – arXiv:2208.03299
- Toolformer: Language Models Can Teach Themselves to Use Tools – arXiv:2302.04761

## 5. Agentic RAG：当检索遇见智能体

Agentic RAG 将检索视为智能体可调用的工具，智能体会根据目标、状态、历史进行规划，动态地组合检索、推理、记忆和操作。

**核心概念**：
- **多步检索规划**：将复杂问题分解为子问题，逐一检索并整合。
- **记忆与反思**：维护短期/长期记忆以支持对话和知识累积。
- **工具生态整合**：连接 SQL 数据库、图谱查询、API、代码执行器等。

**代表性框架/论文**：
- **IRCoT** (见上文)：跨步检索+思维链。
- **ReAct** 范式：基于 LLM 的通用智能体。
- **DSPy**：声明式编程框架，编译器优化 RAG 管道（Stanford, 2023）。
- **Gorilla**：针对 API 调用的大规模微调 (Patil et al., 2023, arXiv:2305.15334)。

## 6. 多模态 RAG 与图 RAG

### 6.1 多模态 RAG

将检索增强从纯文本拓展到图像、表格、音频等多模态数据，例如在视觉问答中检索相关图片，或生成回答时引用表格数据。

**重要工作**：
- **MuRAG**：在多模态知识库上检索图片+文字，辅助多模态生成。  
  - Wenhu Chen et al., 2022, arXiv:2210.02928
- **RA-CM3**：训练一个多模态生成模型，检索外部图文对来增强上下文。  
  - Michihiro Yasunaga et al., 2023, arXiv:2211.12561

### 6.2 图 RAG（Graph RAG）

利用知识图谱或文档层次结构增强检索，捕获实体间的关系，使结果更具全局上下文。微软开源的 GraphRAG 项目（2024）通过构建实体知识图谱并对社区进行摘要，在需要全域理解的数据集上展现优势。

**核心思想**：将文档库转化为图表示，并利用图算法（如社区检测）生成层次化摘要，回答能够覆盖整个主题。

## 7. 评估方法与基准

### 7.1 常见基准

- **KILT**：汇集5个知识密集型任务（事实核查、问答、对话等），要求模型输出附带出处 (Petroni et al., 2021, arXiv:2009.02252)。
- **RGB** (Retrieval-Augmented Generation Benchmark)：评估信息整合、噪声鲁棒性和拒答能力 (Chen et al., 2023, arXiv:2304.09891)。
- **MQAG** (Multi-Question Answer Generation)：覆盖新闻、Wiki等域的多维度评估。

### 7.2 自动评估框架

传统指标（BLEU/ROUGE）难以捕捉事实正确性。新一代评估框架侧重**忠实度、答案相关性、上下文精确性**。

#### RAGAS: Automated Evaluation of Retrieval Augmented Generation
- **来源**: arXiv:2309.15217
- **作者**: Shahul ES et al.
- **链接**: https://arxiv.org/abs/2309.15217
- **核心贡献**: 提供无需人工标注的指标（忠实度、答案相关度、上下文召回率等），已集成进 LangChain 和 LlamaIndex。

#### ARES: An Automated Evaluation Framework for Retrieval-Augmented Generation Systems
- **来源**: arXiv:2311.09476
- **作者**: Jon Saad-Falcon et al.
- **链接**: https://arxiv.org/abs/2311.09476
- **核心贡献**: 通过基于LLM的裁判与少量标注微调，可定制化评估RAG各组件。

**其他工具**：
- **TruLens**：提供可观测性与反馈函数，用于评估RAG应用。
- **DeepEval**：单元测试式的RAG评估。

## 9. 挑战与未来趋势

1. **长上下文与检索协同**：GPT-4 等模型上下文窗口已扩至 128K，但仍存在“Lost in the Middle”现象，因此需要**检索引导的上下文组织**，而非简单堆叠信息。

2. **噪声鲁棒性**：检索结果可能包含矛盾或无关信息，如何训练模型在噪声中做出正确判断是核心难题。Self-RAG、LLM-as-Critic 等范式将继续细化。

3. **可变检索粒度与自适应检索时机**：并非每一句生成都需要检索，自适应决策如何实现最优平衡（如 FLARE、Self-RAG 已初步探索）。

4. **多模态与结构化数据融合**：未来 RAG 需要同时处理文本、表格、图片、图谱等异构数据，并理解其关联（多模态 Retriever 与生成器的联合优化）。

5. **生产环境下的时效性与可维护性**：知识库持续更新，索引策略需支持增量更新；模型输出需严格保证可溯源性与版权合规。

6. **面向多语言的 RAG**：多语言检索器和跨语言生成质量提升亟待补强，尤其低资源语言。

## 10. 结语

RAG 正从单次“检索-阅读”的朴素模式，快速进化为具备自省、规划、工具使用能力的模块化智能体。其技术堆栈涵盖查询增强、混合检索、重排序、上下文压缩、训练范式（联合微调、RAFT、强化学习）以及自动化评估等多个层面。面向未来，RAG 将更深地与多模态、图数据、多轮规划相融合，成为事实可信的智能系统的核心基础设施。
