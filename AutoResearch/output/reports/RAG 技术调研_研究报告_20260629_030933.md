# RAG（检索增强生成）技术综合研究报告

## 摘要
检索增强生成（RAG）已成为大语言模型（LLM）领域应对知识截断、幻觉和垂直领域适配的核心范式。它将检索系统与生成模型深度耦合，在推理时注入外部最新知识，显著提升事实性和领域覆盖。本报告系统梳理 RAG 的定义、发展脉络、核心架构（朴素 RAG、高级 RAG、模块化 RAG）、技术栈（分块、嵌入、检索、重排序、提示组织）、最新研究方向（多模态、图 RAG、主动检索、自我反思、长上下文融合）以及评估体系。所有关键观点均标注可靠来源，并附有 arXiv 编号与链接。

---

## 1. 概念与背景

**定义**  
RAG 是一种“检索 + 生成”的混合框架：给定用户查询，系统首先从外部知识库（如文档集、数据库、网络）检索最相关的信息片段，然后将其作为上下文拼接到提示（prompt）中，交由 LLM 生成最终回答。  
*来源：Lewis et al., 2020; Gao et al., 2023*

**核心动机**  
- **补足 LLM 的静态知识**：模型训练数据存在截止日期，RAG 动态接入最新知识。  
- **缓解幻觉**：显式引用来源，增强事实一致性。  
- **可解释性与归因**：回答可溯源至具体文档。  
- **知识密集任务优化**：如开放域问答、事实验证、临床决策等。

**关键论文**  
```
#### Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks
- **来源**: arXiv:2005.11401 (2020)  
- **作者**: Patrick Lewis et al. (Facebook AI Research)  
- **链接**: https://arxiv.org/abs/2005.11401  
- **核心贡献**: 提出 RAG 概念，将预训练参数记忆与非参数记忆（维基百科稠密索引）结合，在知识密集型 NLP 任务上取得当时最佳成绩。
```

## 2. 范式演进：从朴素到模块化

### 2.1 朴素 RAG
经典“检索-阅读”流水线：单次检索后，直接将 top-k 文档拼接喂给生成器。  
特点：结构简单，但存在检索质量敏感、上下文冗余、无法处理多跳推理等缺点。

### 2.2 高级 RAG
在朴素基础上增加**检索前预处理**和**检索后优化**，形成更精细的信息流。  
- **检索前**：查询重写（Query2doc, HyDE, Step-back prompting）、子问题分解。  
- **检索后**：重排序（Cross-encoder reranker）、上下文压缩、Summarization。  
*来源：Gao et al., 2023*

### 2.3 模块化 RAG
将 RAG 拆解为可组合的功能模块（检索、记忆、融合、路由、预测等），支持根据任务灵活编排。  
例如：**路由模块**根据查询复杂度选择不同检索策略；**预训练时融入检索**（RETRO）、**生成时主动激活检索**（Self-RAG）。  
*来源：Gao et al., 2023；Li et al., 2024*

**综述与范式划分核心文献**

```
- **核心贡献**: 系统梳理 RAG 发展，提出朴素 RAG、高级 RAG、模块化 RAG 三阶段分类法，评估体系与技术树。
```

## 3. 技术栈深度解析

### 3.1 索引与分块策略
- **分块大小**：平衡语义完整性与检索粒度，常用 256~512 tokens，针对长文档可混合大小。  
- **元数据增强**：加入标题、时间、来源权重，提高检索相关度。  
- **多级索引**：摘要索引 + 细粒度块索引（如 LlamaIndex 的 Summary Index）以支持不同查询粒度。

### 3.2 嵌入模型与向量数据库
- **嵌入模型**：从早期 DPR (Dense Passage Retrieval) 到如今的 BGE, E5, Instructor, Jina Embeddings；多语言嵌入支持。  
- **向量数据库**：FAISS, Milvus, Weaviate, Chroma, Qdrant 等；核心关注近似最近邻搜索（ANN）效率、混合检索（稠密+稀疏，如 BM25）支持。

### 3.3 检索与重排序
- **混合检索**：结合稠密向量语义匹配与关键词（BM25）精确匹配，互补优势。  
- **多阶段检索**：第一阶段粗筛（向量检索 top-100），第二阶段精排（Cross-encoder，如 Cohere Rerank, BGE Reranker）。  
*论文参考*：DPR (Karpukhin et al., 2020, arXiv:2004.04906), ColBERT v2, LLM as a reranker (RankGPT, Sun et al., 2023, arXiv:2304.09542)。

### 3.4 上下文组织与提示工程
- **文档拼接顺序**：按相关度降序，相似文档间隔排列避免“Lost in the Middle”现象。  
- **上下文压缩**：仅保留文档中与查询相关的片段（LLMLingua, LongLLMLingua）。  
- **引用注入**：要求模型在回答中指出引用来源（如“[1]”），便于事实核查。  
*来源：Liu et al., 2023 "Lost in the Middle", arXiv:2307.03172*

```
#### Lost in the Middle: How Language Models Use Long Contexts
- **来源**: arXiv:2307.03172 (2023)  
- **作者**: Nelson F. Liu et al.  
- **链接**: https://arxiv.org/abs/2307.03172  
- **核心贡献**: 发现 LLM 对长文本中间位置的信息利用最弱，影响 RAG 文档排列策略。
```

## 4. 前沿研究方向

### 4.1 主动检索与自适应 RAG
- **Self-RAG**：模型在生成过程中自主判断是否检索、检索什么，并对检索结果进行反思与选择，以提升事实性和可归因性。  
- **CRAG (Corrective RAG)**：动态评估检索文档质量，质量差时触发网络搜索等补充检索，并过滤不相关内容。  
- **FLARE / IRCoT**：让 LLM 预测下一步所需信息，进行前瞻性主动检索。

```
#### Self-RAG: Learning to Retrieve, Generate, and Critique through Self-Reflection
- **来源**: arXiv:2310.11511 (2023)  
- **作者**: Akari Asai et al.  
- **链接**: https://arxiv.org/abs/2310.11511  
- **核心贡献**: 训练 LLM 通过特殊标记（reflection tokens）判断是否检索、检索结果相关性及生成质量，形成自我评估闭环。
```

```
#### Corrective Retrieval Augmented Generation
- **来源**: arXiv:2401.15884 (2024)  
- **作者**: Shi-Qi Yan et al.  
- **链接**: https://arxiv.org/abs/2401.15884  
- **核心贡献**: 提出检索评估器，对低质检索结果自动启动网络搜索，并通过“知识精炼”算法进一步提升文档利用率。
```

### 4.2 图 RAG（Graph RAG）
将实体关系图谱引入 RAG，支持多跳推理和全局摘要。  
**Microsoft GraphRAG**：利用 LLM 从原始文档构建知识图谱，生成社区摘要，回答全局性汇总问题时效果显著。  
**GNN-RAG**：将图神经网络推理与 LLM 结合用于知识图谱问答。

```
#### From Local to Global: A Graph RAG Approach to Query-Focused Summarization
- **来源**: arXiv:2404.16130 (2024)  
- **作者**: Darren Edge et al. (Microsoft)  
- **链接**: https://arxiv.org/abs/2404.16130  
- **核心贡献**: 提出 Graph RAG 流程，通过实体提取、图推理和社区摘要生成，实现对整个语料库的全局理解。
```

### 4.3 多模态 RAG
扩展检索源至图像、音频、视频，跨模态对齐信息。  
- **MuRAG**：多模态 RAG，可基于检索到的原始图像和文本生成多模态答案。  
- **RA-CM3**：同时检索并生成图像和文本，实现多模态内容创作。

```
#### MuRAG: Multimodal Retrieval-Augmented Generator for Open Question Answering over Images and Text
- **来源**: arXiv:2210.02928 (2022)  
- **作者**: Wenhu Chen et al.  
- **链接**: https://arxiv.org/abs/2210.02928  
- **核心贡献**: 构建多模态知识库，利用 ViT 和 T5 联合检索与生成，支持图像和文本混合检索。
```

### 4.4 长上下文与 RAG 的融合
随着 LLM 上下文窗口突破百万 token（如 Gemini, GPT-4-turbo），RAG 与长上下文的互补成为焦点。  
- **LONGPAD**：长上下文 + 检索并行，实现长文档高效问答。  
- 实践观察：长上下文可容纳更多检索文档，但仍需精心设计排序和位置管理；RAG 在处理大规模外部知识时仍具有成本效益。

### 4.5 检索增强的推理与规划
- **REPLUG**：将检索器作为组件插入 LLM，支持多文档集成推理。  
- **IRCoT**：交织检索与链式思维（CoT），逐步检索并推理。  
- **Demonstrate–Search–Predict (DSP)**：通过编写检索-推理流水线实现复杂任务（如多跳问答）。

```
#### ReAct: Synergizing Reasoning and Acting in Language Models
- **来源**: arXiv:2210.03629 (2022)  
- **作者**: Shunyu Yao et al.  
- **链接**: https://arxiv.org/abs/2210.03629  
- **核心贡献**: 将推理轨迹与行动（如检索）交替进行，极大提升知识密集型任务效果。
```

## 5. 训练与微调技巧

RAG 模式不仅用于推理，越来越多的工作将其融入训练阶段，增强模型内在检索感知能力。

- **SAIL** (Self-Augmented In-Context Learning)：利用 LLM 自生成相关文档，模拟检索增强过程进行无数据微调。  
- **RA-DIT**：分别微调检索器与 LLM，使其相互适配，提升 RAG 流水线整体效果。  
- **RAT (Retrieval Augmented Training)** ：在预训练或指令微调中引入检索文档，让模型学会如何利用上下文。  
- **Recitation-Augmented Generation**：在生成前先让模型“背诵”相关事实作为前置思考。

```
#### RA-DIT: Retrieval-Augmented Dual Instruction Tuning
- **来源**: arXiv:2310.01352 (2023)  
- **作者**: Xi Victoria Lin et al. (Meta AI)  
- **链接**: https://arxiv.org/abs/2310.01352  
- **核心贡献**: 提出双指令微调方案，分别优化 LLM 利用检索结果的能力，以及检索器理解 LLM 信息偏好的能力。
```

## 6. 评估与基准

全面评估 RAG 系统需同时考量检索质量、生成质量及两者协同效果。

- **RGB (Retrieval, Generation, Benchmarking)** ：首套系统评估 LLM 在 RAG 场景下表现的基准，涵盖噪声鲁棒性、拒答能力、多文档集成等维度。  
- **RAGAS**：自动化评估框架，运用忠实度、答案相关性、上下文相关性等指标无需人工标注。  
- **ALCE (Automatic LLMs’ Citation Evaluation)** ：评估模型自动引用能力，关注引用的准确性和完整性。  
- **KILT**：知识密集型任务统一基准，涵盖事实核查、对话、QA 等，要求可溯源。

```
#### Benchmarking Large Language Models in Retrieval-Augmented Generation
- **来源**: arXiv:2309.01431 (2023)  
- **作者**: Jiawei Chen et al.  
- **链接**: https://arxiv.org/abs/2309.01431  
- **核心贡献**: 构建 RGB 基准，测试 LLM 对检索信息的利用、抗噪声和拒答能力，揭示 RAG 的鲁棒性短板。
```

```
#### RAGAS: Automated Evaluation of Retrieval Augmented Generation
- **来源**: arXiv:2309.15217 (2023)  
- **作者**: Shahul Es et al.
```

## 8. 挑战与未来展望

尽管 RAG 成效显著，仍面临诸多挑战，也是近期研究热点：

1. **检索质量的鲁棒性**：噪声文档易引入错误信息，需更强容错机制（如 CRAG, Self-RAG）。  
2. **多跳推理与信息拼接**：复杂查询需要跨文档推理，图 RAG 和交织检索推理正试图解决。  
3. **成本与延迟**：多阶段检索、重排序和拼接会影响实时性，蒸馏小模型、缓存策略是缓解方向。  
4. **领域适配**：金融、医疗等垂直领域常缺乏标注数据，SAIL 等自增强微调方法开始缓解。  
5. **安全与隐私**：外部知识库可能含有偏见或敏感信息，带来合规风险。  
6. **长上下文与 RAG 的边界**：何时该检索？何时直接依赖长上下文？正在成为开放研究问题。  
7. **多模态、多语言扩展**：从文本走向多模态，从高资源语言走向低资源语言。

---

## 9. 结论

RAG 已构建出一套“检索-增强-生成”的完整技术生态，从最初的线性流水线演化至可自我反思、主动检索、多模态融合的复杂系统。未来方向将更强调 **“何时检索”、“如何精炼”** 与 **“安全可信”** ，并与长上下文模型、智能体（Agent）范式深度整合。研究者与实践者可依据本报告梳理的关键论文和框架快速切入前沿。
